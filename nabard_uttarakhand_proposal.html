1. Executive Summary

Uttarakhand's financial and cooperative banking ecosystems are undergoing a rapid digital evolution. Under the national mandate for Primary Agricultural Cooperative Societies (PACS) computerization and the expansion of digital banking services by District Central Cooperative Banks (DCCBs) and Regional Rural Banks (such as Uttarakhand Gramin Bank), IT infrastructure has spread deeply into the remote, mountainous districts of the state.

However, this rapid decentralization poses severe operational and security challenges:

  • Security & Vulnerability Exposure: Maintaining rigid compliance with the Reserve Bank of India (RBI) cybersecurity guidelines for cooperative banks is extremely difficult in remote mountain branches without dedicated IT staff.
  • Asset Visibility Gaps: Physical tracking of distributed hardware (computers, routers, biometric scanners, solar power systems, and CCTV units) across hundreds of rural sites is prone to leakage, obsolescence, and theft.
  • Delayed Maintenance: Technical breakdowns in hilly regions like Pithoragarh, Chamoli, or Uttarkashi lead to extended system downtime, disrupting banking services for rural farmers.

The Solution: QS Asset

QS Asset is a premium, secure, and enterprise-grade IT Asset Management (ITAM), Network Discovery, and ITIL-compliant Service Desk platform. Built to run either on-premises (inside secure, air-gapped banking data centers) or in the cloud, QS Asset integrates automated Layer-2/3 network sweeping, remote agentless hardware audits, continuous endpoint compliance telemetry, and a fully featured ticketing and maintenance engine into a unified interface.

This proposal outlines the strategic deployment of the QS Asset Platform to fortify IT security, automate asset tracking, and optimize IT operations for NABARD Uttarakhand and the cooperative/regional banking sectors it supervises and funds.

2. Project Objectives

  1. Achieve 100% IT Inventory Visibility: Automatically discover, catalog, and monitor every device (authorized or unauthorized) connected to the local subnets of the NABARD Regional Office, DCCBs, and PACS.
  2. Enforce Banking Cybersecurity Compliance: Enable automatic tracking of operating system patch levels, firewall states, antivirus running status, and software license compliance to satisfy RBI audits.
  3. Minimize Banking Downtime in Hilly Terrains: Establish an ITIL-compliant service desk with SLA policies and automated work-order dispatching for field engineers.
  4. Prevent Asset Leakage & Hardware Loss: Implement a digitized asset-checkout lifecycle and a localized "Asset Attestation" mechanism to physically verify hardware presence at remote locations.
  5. Optimize IT Expenditure: Cleanly audit software licensing usage and track Annual Maintenance Contracts (AMCs) and warranties to eliminate vendor overcharging.
Security Focus
On-Premises Air-Gapped Deployment: The entire platform can be deployed in a localized virtual machine environment (via Docker Compose or bare-metal node runtimes) behind the bank's internal firewalls. Absolutely no telemetries or system attributes are transmitted externally, fully satisfying banking data sovereignty rules.

3. Platform Architecture

QS Asset employs a hybrid-capable architecture. When deployed on-premises within a secure banking network or inside the State Cooperative Bank's central data center, it functions without external dependencies:

Centralized Core (Data Center / Cloud)

  • Next.js Frontend: Fast, modern dashboard running securely on Port 3100.
  • NestJS Backend API: High-performance business logic orchestrating scans on Port 4100.
  • PostgreSQL Database: Advanced relational schema storing CMDB, tickets, and contracts.
  • Redis Task Queue: Handles asynchronous network discovery tasks smoothly.

Subnet Sweepers & Local Agents

  • Agentless Sweeper: Runs Layer-2 ARP and Layer-3 ICMP active sweeps across subnets.
  • SSH / WinRM Poller: Queries remote terminals for CPU, OS, RAM, and Disk details.
  • Continuous Telemetry Agent: Lightweight daemon running on local bank workstations.

4. Comprehensive Feature Suite

A. Intelligent Network Discovery & Active Sweeper

The platform initiates network audits without manual interference. Using active ARP and ping sweeps, it registers active hosts inside subnets. It securely logs into terminal interfaces via SSH and WinRM protocols to gather low-level CPU, RAM, and storage specifications. A continuous background agent reports delta telemetry parameters periodically to catch any changes.

B. Enterprise Configuration Management Database (CMDB)

Maintains dynamic records of hardware and software dependencies. Out-of-the-box asset typologies allow bank administrators to register everything from core servers to branch CCTV cameras and solar systems. It continuously records endpoint security postures, including BitLocker/FileVault encryption status, active local firewalls, and antivirus definition updates.

C. ITIL-Compliant Service Desk (ITSM)

Integrates ticketing, incidents, changes, and work orders with physical CMDB records. When a router fails, a support ticket is generated and pre-linked with the router's warranty, configuration backups, and switch port history. Field engineers are dispatched with complete hardware details, diagnostic files, and checksheets, significantly boosting first-visit fix rates.

D. Compliance, Procurement & Financial Management

Mitigates operational overhead and license overspending. Tracks OS updates and patch levels to satisfy standard security checks. Compares actual software installations against cataloged commercial licenses. Automatically triggers support warnings and email alerts weeks before hardware warranties and AMCs expire.

5. Unique Selling Propositions (USPs) for Uttarakhand

1. Hill-Network Optimization (Low-Bandwidth Operation)

Hilly regions in Uttarakhand face erratic satellite/VSAT bandwidth, high latency, and frequent fiber cuts. The QS Asset continuous agent is built with a resilient, state-aware telemetry loop. Telemetry updates are packaged into highly compressed, differential JSON payloads. If a remote branch loses internet connectivity for days, the agent caches execution history locally and syncs seamlessly once connectivity is restored, without duplicate packets or network strain.

2. Compliance with RBI Circulars for Cooperative & Rural Banks

Cooperative banks are under strict RBI scrutiny to implement cyber security controls (e.g., preventing unauthorized devices from accessing the Core Banking network). QS Asset’s active network sweep immediately flags any "rogue" device (e.g., a personal laptop plugged into a banking switch port) that does not match the CMDB whitelist. It fires immediate SMS/Email alerts and logs audit trails, keeping the bank audit-ready 24/7.

3. Digital PACS Asset Accountability (Anti-Leakage Control)

Under the Computerization of PACS project, hundreds of rural cooperative societies receive expensive computer setups. Physically tracking their operating status and ensuring they are not misplaced is a massive administrative headache. With QS Asset's "Asset Attestation" module, local PACS administrators can complete a single-click attestation from their portal. Additionally, the network sweeps continuously map local subnets to ensure the machines remain physically located at the registered coordinates.

4. WMI & SSH Push-to-Deploy Simplicity

Remote branches do not have technical personnel to manually install monitoring software. Through a single central console, an administrator can utilize the visual "Remote Push Deployer" to automatically transmit, configure, and register the QS Agent service across thousands of target machines within a subnet using existing domain or network credentials.

7. Strategic Implementation Roadmap

A detailed 16-week rollout strategy tailored for Uttarakhand's geographic layout:

Weeks 01 - 04: Central Installation & Core Configurations
Provisioning servers inside secure banking hosting / Data Center. Database setup, tenant creation, and Active Directory authentication sync. Defining Core Asset Types (workstations, servers, switches, routers, solar units).
Weeks 05 - 08: Pilot Subnet Sweeps & Vault Integrations
Initiating agentless network sweeps at HQ and Dehradun Regional Office. Setting up secure credential Vaults for SSH, SNMP, and WinRM tasks. Launching pilot agent installations on 100 client machines.
Weeks 09 - 12: Mass Regional Branch Expansion & GPO Deployment
Pushing mass agent installations using Active Directory Group Policy (GPO). Initiating remote subnet sweeps for District Cooperative Banks (DCCBs). Deploying the technician service desk portal and training support agents.
Weeks 13 - 16: Attestation Framework & Final Handover
Initializing PACS Asset Attestation workflow for pilot rural districts. Setting up automated compliance reporting schedules (RBI layout format). Continuous operations training and final handovers to the IT team.

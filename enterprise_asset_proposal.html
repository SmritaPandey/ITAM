01Executive summary

Distributed estates — endpoints, network gear, facilities equipment, cameras, fleet, and cloud resources — still live in spreadsheets and siloed tools. That creates blind inventory, slow incident triage, software audit risk, and expired contracts that become emergency spend.

QS Assets (product of NeurQ AI Labs Private Limited) is a production platform that unifies agent + agentless discovery, a living asset / CMDB record, ITIL-style service desk, monitoring, vulnerability & patch workflows, and non-IT modules (CCTV, fleet, facility) in one control plane. Customers can run managed SaaS at www.qsasset.com or self-host with Docker and PostgreSQL when residency requires it.

Typical enterprise pains

  • Blind spots: Manual tracking leaves large shares of network assets unmonitored.
  • Technician friction: Tickets lack hardware / patch context, extending MTTR.
  • License liability: Weak SAM invites audit exposure and shelfware spend.
  • Contract oversights: Expired AMC / warranty triggers emergency repair cost.

QS Assets outcomes

  • Auto discovery: Agent + SNMP / SSH / WMI / nmap sweeps keep inventory alive.
  • ITSM + CMDB: Incidents open against the same asset record technicians need.

07Implementation roadmap

Phased rollout designed for controlled cutover. SaaS customers typically compress Phase 1 significantly.

Weeks 1–2 · Foundation

Tenant / environment stand-up (SaaS or Docker). SSO or directory hooks. Role model. Sites & departments. Branding and alert channels.

Weeks 3–5 · Discovery pilot

Credential vault for SNMP / SSH / WMI. Agentless sweep of approved CIDRs. Pilot agent rollout (~50–100 endpoints). Discovery → managed asset approvals.

Weeks 6–8 · Service desk & scale-out

ITSM queues, SLA policies, portal users. Broader agent deployment via GPO / MDM / packaging. Patch & vulnerability workflows enabled.

Weeks 9–12 · Harden & handover

Automation rules, contract / AMC alarms, attestation cycles, report schedules. Admin training, runbooks, go-live checklist, hypercare.
